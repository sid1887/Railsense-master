## Components Created

### 1. **HaltStatusCard.tsx** (220 lines)
**Purpose:** Display current halt status with visual severity indicators
**Features:**
- Color-coded status (red for halt, green for moving)
- Duration tracking and display
- Severity level dots (critical, high, medium, low)
- Pulsing alert animations on halt detection
- Custom hook: `useHaltAnimation()` for severity pulsing
- Responsive card layout

**Data Bindings:**
```tsx
<HaltStatusCard
  haltDetection={insightData.haltDetection}
  trainDelay={insightData.trainData.delay}
/>
```

**Visual States:**
- ✅ Moving: Green checkmark + "Moving normally"
- ⚠️ Halted: Red alert + duration + reason + severity

### 2. **UncertaintyGauge.tsx** (240 lines)

- Circular SVG gauge with smooth fill animation
- Color gradient from green (low uncertainty) to red (critical)
- Emoji indicators (😊😐😟😰) mapped to uncertainty levels
- Weighted factor breakdown display:

**Data Bindings:**
```tsx

// uncertainty = { score: 0-100, level: 'LOW'|'MEDIUM'|'HIGH'|'CRITICAL', factors: {...} }

**Algorithm Visualization:**
Displays real-time calculation: Score = (halt×0.4) + (traffic×0.35) + (weather×0.25)

- Dark blue (low density) → Orange (medium) → Red (critical)
- Grid square tooltips showing intensity percentage
- Train count statistics

- Real-time updates as trains move

**Data Bindings:**

```

**Heatmap Algorithm:**
- Grid centered on train location
- 10km × 10km area
- Each cell: 1.25km × 1.25km
- Intensity = trains within cell + nearby cells / total nearby trains
- Color mapped to intensity

**Color Scale:**
- 🟦 Low (0-20%): Dark blue
- 🟩 Low-Med (20-40%): Light blue
- 🟨 Medium (40-60%): Yellow
- 🟧 High (60-80%): Orange

## Integration Architecture

All 8 components are integrated into the Train Detail Page (`/app/train/[trainNumber]/page.tsx`):

```
         Train Detail Page (/app/train/[trainNumber]/page.tsx)
                  ↓ [uses hook]
              useTrainData()
                  ↓ [fetches]
          POST /api/train-details
                  ↓ [orchestrates]
            orchestrator.ts
                  ↓ [calls services]
        ┌────────┼────────┬────────┬────────┐
        ↓        ↓        ↓        ↓        ↓
   haltDetection  trafficAnalyzer  predictionEngine  insightGenerator
        ↓        ↓        ↓        ↓        ↓
    Complex     Traffic  Wait    Passenger
    Analysis    Analysis  Prediction Insights
        ↓        ↓        ↓        ↓        ↓
    └────────┬────────┬────────┬────────┘
             ↓
      TrainInsightData object
        ├─ trainData
        ├─ haltDetection
        ├─ trafficAnalysis
        ├─ prediction
        ├─ uncertainty
        └─ insight
             ↓
    [Distributed to 8 Components]
    ├─ HaltStatusCard (haltDetection)
    ├─ RouteTimeline (trainData.stations)
    ├─ LiveTrainMap (trainData.location)
    ├─ CongestionHeatmap (trafficAnalysis)
    ├─ UncertaintyGauge (uncertainty)
    ├─ TrafficIndicator (trafficAnalysis)
    ├─ PredictionCard (prediction)
    └─ InsightPanel (insight)
```

## Animation Patterns

All components use consistent Framer Motion animation patterns:

### Container Reveal (All Components)
```tsx
const containerVariants = {
  hidden: { opacity: 0, y: 20 },
  visible: {
    opacity: 1,
    y: 0,
    transition: { duration: 0.5, ease: "easeOut" }
  }
};
```

### Element Stagger (UncertaintyGauge, InsightPanel)
```tsx
const itemVariants = {
  hidden: { opacity: 0, x: -10 },
  visible: {
    opacity: 1,
    x: 0,
    transition: { duration: 0.3 }
  }
};

<motion.div variants={containerVariants} initial="hidden" animate="visible">
  {items.map((item, i) => (
    <motion.div key={i} variants={itemVariants} custom={i} transition={{ delay: i * 0.1 }}>
      {item}
    </motion.div>
  ))}
</motion.div>
```

### Pulsing Effect (HaltStatusCard Alert)
```tsx
<motion.div
  animate={{ scale: [1, 1.05, 1] }}
  transition={{ repeat: Infinity, duration: 2 }}
>
  {/* Important value */}
</motion.div>
```

### Progress Fill (UncertaintyGauge, PredictionCard)
```tsx
<motion.div
  initial={{ width: 0 }}
  animate={{ width: `${percentage}%` }}
  transition={{ duration: 1, delay: 0.2 }}
/>
```

### Scale on Hover (Cards)
```tsx
<motion.div
  whileHover={{ scale: 1.02 }}
  transition={{ type: "spring", stiffness: 300, damping: 20 }}
>
```

## Real-Time Data Flow

Components receive live data through the following flow:

1. **Page Load:** useTrainData hook fetches from `/api/train-details`
2. **Polling:** Hook auto-refetches every 5 seconds (configurable)
3. **Data Update:** insightData state updates with new TrainInsightData
4. **Component Re-render:** All 8 components receive updated props
5. **Animation Trigger:** Framer Motion detects data changes and animates updates
6. **Visual Update:** User sees smooth animated transitions in all displays

**Example Timeline:**
```
T=0s   → Page loads, spinner shown
T=0.2s → Data arrives, components animate in (staggered 0.1s each)
T=5s   → Auto-refetch triggers
T=5.1s → Data updates propagate to components
T=5.2s → Animations update (gauge fills, bars slide, etc.)
T=10s  → Next auto-refetch
```
