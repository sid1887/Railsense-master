INTELLIGENCE SUBSYSTEM DESIGN METHODOLOGY
───────────────────────────────────────────

SERVICE 1: Network Intelligence Engine
    Purpose: Multi-train interaction awareness
    Analysis Range: 50km radius from current train
    Features:
        • Identify nearby trains (within range)
        • Classify interactions (same direction, opposite, crossing)
        • Compute congestion score (0-100 scale)
        • Assess relative speeds and collision risk
    Output: NearbyTrains[], CongestionLevel, CongestionScore

SERVICE 2: Dwell Time Prediction
    Purpose: Forecast station stop duration
    Input Factors:
        • Station type (junction adds 2+ minutes)
        • Current delay (accumulation)
        • Network congestion level (propagates to dwell)
        • Nearby trains competing for same platform
    Calculation:
        • Base dwell: 3 minutes
        • Junction penalty: +2 minutes
        • High delay: +1-5 minutes
        • Congestion: +0-3 minutes
        • Multiple trains: +1 minute
    Output: ExpectedDwellTime (integer), DwellRisk (low/medium/high), Reasons[]

SERVICE 3: Crossing Detection
    Purpose: Identify collision risk with opposing trains
    Detection Logic:
        • Check if opposing train on same track
        • Calculate distance and relative speed
        • Compute time-to-conflict: (distance / combined_speed) * 60
        • Classify risk level based on time window
    Risk Levels:
        • CRITICAL: <5 minutes to potential conflict
        • HIGH: 5-10 minutes
        • MEDIUM: 10-20 minutes
        • LOW: >20 minutes or no conflict
    Output: HasOpposingTrain, Distance, TimeToConflict, RiskLevel

SERVICE 4: Platform Occupancy Analysis
    Purpose: Forecast platform availability and passenger wait times
    Analysis:
        • Count trains scheduled to arrive in next 30 minutes
        • Calculate platform load: 30% base + 20% per incoming train
        • Determine waiting probability: load/100
        • Estimate wait time: probability * 15 minutes (maximum)
    Output: PlatformLoad (0-100%), WaitingProbability (0-1.0), ExpectedWaitTime (minutes)

SERVICE 5: Advanced ETA Prediction (V2 Engine)
    Purpose: Multi-factor arrival time forecasting
    Input Synthesis:
        • Base train delay from current position
        • Dwell prediction at next station
        • Congestion penalty across network
        • Crossing delay if opposing trains present
        • Platform wait time if occupancy high
    ETA Calculation:
        ETA = ScheduledTime + (CurrentDelay × 0.8) + DwellRisk(0-3m) + CongestionPenalty(0-5m)
              + CrossingDelay(0-10m) + PlatformWait(0-15m)

    Confidence Formula:
        BaseConfidence = 70%
        If(liveGPSAvailable) += 15%
        If(shortDwellPredicted) += 10%
        If(lowCongestion) += 5%
        FinalConfidence = min(100%, BaseConfidence + Bonuses)

    Risk Level Assignment:
        • CRITICAL: TotalDelay > 30 minutes
        • HIGH: TotalDelay > 20 minutes
        • MEDIUM: TotalDelay > 10 minutes
        • LOW: TotalDelay ≤ 10 minutes

DATA QUALITY TRACKING
─────────────────────

Each response includes transparency metrics:
    • liveGPS: Boolean (real GPS available)
    • stationMapping: Boolean (station correctly identified)
    • networkAwareness: Boolean (nearby trains analyzed)
    • predictionStrength: Integer 0-100 (confidence factor)
    • liveUnavailable: Boolean (if true, confidence already reduced)
    • sources: String[] (array of data providers used)

Users can see EXACTLY which data is real vs estimated.

    Export: analyzeNearbyTrains(trainNumber, location, stations, nearbyTrains)

    Analyzes:

    • Direction classification (same/opposite/crossing)
    • Relative speed calculations
    • Congestion density scoring

    Output: NetworkIntelligenceResult

        ├─ congestionLevel: 'low'|'medium'|'high'|'severe'
        └─ congestionScore: 0-100

SERVICE: dwellPredictionService.ts (80 lines)
    Export: predictDwell(trainNumber, baseData, networkData, currentDelay)

    Factors:
    • Base dwell: 3 minutes
    • Junction station: +2 minutes
    • Delay accumulation: +1-5 minutes
    • Network congestion: +0-3 minutes
    • Multiple nearby trains: +1 minute

    Output: DwellPrediction
        ├─ expectedDwellTime: number (minutes)
        ├─ dwellRisk: 'low'|'medium'|'high'
        └─ reasons: string[] (human-readable factors)

SERVICE: crossingDetectionService.ts (70 lines)
    Export: detectCrossing(trainNumber, location, nearbyTrains)

    Detection:
    • Find opposing trains on same track
    • Calculate distance in kilometers
    • Compute time-to-conflict: (distance / (speed1 + speed2)) * 60 = minutes
    • Classify risk: critical(0-5m), high(5-10m), medium(10-20m), low(>20m)

    Output: CrossingRiskResult
        ├─ hasOpposingTrain: boolean
        ├─ distance: number (km)
        ├─ timeToConflict: number (minutes)
        └─ riskLevel: 'low'|'medium'|'high'|'critical'

SERVICE: platformOccupancyService.ts (90 lines)
    Export: analyzeOccupancy(stationCode, stationIndex, allStations, nearbyTrains)

    Calculation:
    • Base platform load: 30%
    • Add 20% per train arriving in next 30 minutes
    • Cap at 100%
    • Wait probability: load / 100
    • Expected wait: probability × 15 minutes

    Output: PlatformOccupancyResult
        ├─ platformLoad: 0-100% (integer)
        ├─ waitingProbability: 0-1.0 (decimal)
        └─ expectedWaitTime: number (minutes)

SERVICE: predictionEngineV2.ts (120 lines)
    Export: predictArrival(trainNumber, baseData, liveData, dwellPrediction,
                          crossingRisk, platformOccupancy, networkData)

    ETA Formula:
    ETA = ScheduledTime + DelayPropagation + DwellRisk + CongestionPenalty
          + CrossingDelay + PlatformWait

    Factor Breakdowns:
    • delayPropagation: currentDelay × 0.80 (80% carries forward)
    • dwellRisk: 0-3 minutes (from dwellPredictionService)
    • congestionPenalty: {severe:5m, high:3m, medium:1.5m, low:0m}
    • crossingDelay: {critical:10m, high:5m, medium:2m, low:0m}
    • platformWait: occupancy.expectedWaitTime

    Confidence Formula:
    baseConfidence = 70%
    +15% if liveGPS.confidence > 0.7
    +10% if dwellTime < 5 minutes
    +5% if congestionLevel == 'low'
    finalConfidence = min(100%, baseConfidence + adjustments)

    Output: PredictionResult
        ├─ eta: ISO timestamp (estimated arrival)
        ├─ delayForecast: number (minutes from schedule)
        ├─ confidence: 0-100% (prediction certainty)
        ├─ riskLevel: 'low'|'medium'|'high'|'critical'
        └─ factors: {delayPropagation, dwellRisk, congestionPenalty, crossingDelay, platformWait}
